Dumpster Dive

Game Link: https://calrsdr.itch.io/dumpster-dive

This game was our submission to the Pirate Software Game Jam 15.

Prior to the game jam, we were already planning to make a game about interacting with shadows, so when the theme came out and it was "Shadows and Alchemy", we were pleasantly surprised that we already had a base to work off of.

We interpreted shadows as a mechanic where the player needs to stay in shadows to recover health and staying too long in the light will start to cause damage. We interpreted alchemy as a crafting mechanic, where enemies would drop scraps of junk and the player could craft different weapons or items based on a combination of the drops.

I was responsible for all 3D models, animations, the environment, and props. My friends at school who I was working with were responsible for programming the combat, movement, menus, crafting, and weapons.

For the environment, we settled on a dingy sewer where the player, a raccoon, would be trying to escape. Trash has alchemized into monsters and the player needs to defeat them and craft their way out of the sewer. Below are the models and animations made for the project, featuring a protagonist, 4 basic enemies, and a boss.


The process of making 3D models for the game jam was incredibly difficult. Even though the game jam was 2 weeks long, working on all the models and animations for the project alone was incredibly challenging and I barely finished the models on the day of the submission. Unfortunately, with the amount of bugs and lack of experience properly scoping the project, we were unable to submit on time.

The most time consuming part of the project was animation, with this being my first time trying animation. Apart from the raccoon, who's animations were taken from Mixamo, I was responsible for making custom animations for enemies. However, this process was satisfying and I definitely want to spend more time learning to properly animate in the future without such a strict time frame.

While we were unable to submit the game this time, we think the ideas are valuable to work off of and we hope to expand the gameplay and fix bugs while refactoring some of the rushed aspects under the hood. In particular, the light detection mechanic is very finicky and we want to find a way to make it more stable, alongside giving the game a more stylized and focus artistic direction. We also struggled using GitHub for version control as we neglected to learn the basics prior to the jam. This ended up causing several problems with us having to completely remake some mechanics due to an inability to solve merge conflicts. However, we gained a lot of experience from this jam just having to finish a project in a strict timeframe, especially since the jam requires a Game Design Document. The process was most importantly fun and a good learning experience for both me making animations and supporting my friends as they participate in their first game jam. We also got fairly familiar with the quirks of Godot, which definitely helped us for our next game jam where we were able to submit on time. After the jam, we also learned to use GitHub so that future projects would run smoothly. As a minimum viable product, we're overall happy with how it turned out and it serves as a good base for what feels to be promising mechanics.

For more information on design specifics, refer to the Game Design Document attached in the description of the itch.io page.
